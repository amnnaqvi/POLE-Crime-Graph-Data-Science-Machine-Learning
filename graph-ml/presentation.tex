\begin{frame}{Problem Statement}
  How can a POLE crime graph help investigators decide where to look next, without pretending the model can prove guilt?

  \vspace{0.3cm}
  \begin{columns}[T]
    \column{0.48\textwidth}
      \textbf{Initial modelling idea}
      \begin{itemize}
        \item Predict \tinycode{Person -> Crime}.
        \item Treat \tinycode{PARTY\_TO} as the main machine-learning (ML) target.
        \item Write back likely person-crime links: save model guesses as new graph relationships.
      \end{itemize}

    \column{0.48\textwidth}
      \textbf{Final modelling direction}
      \begin{itemize}
        \item Find hotspots and repeat place patterns.
        \item Detect social communities and review-priority people.
        \item Use graph machine learning (GML) for review-only social-link candidates.
        \item Do not create automated person-crime claims.
      \end{itemize}
  \end{columns}
\end{frame}

\begin{frame}[fragile]{Key Stats Query}
  \small
  Purpose: check whether person-crime prediction has enough labelled evidence to be fair or useful.

  \vspace{0.2cm}
  \begin{verbatim}
MATCH (p:Person), (c:Crime)
WITH count(DISTINCT p) AS persons, count(DISTINCT c) AS crimes
MATCH (:Person)-[r:PARTY_TO]->(:Crime)
RETURN persons, crimes, count(r) AS observedPartyTo,
       persons * crimes AS possiblePairs,
       100.0 * count(r) / (persons * crimes) AS positivePercent;
  \end{verbatim}

  \vspace{0.1cm}
  Output: only 55 observed \tinycode{PARTY\_TO} links out of 10,613,178 possible pairs, or 0.0005\%.
\end{frame}

\begin{frame}{Graph ML Design}
  \textbf{Final GML target:} Link prediction (LP), meaning prediction of missing relationships between existing nodes. Here, the model scores pairs of people who do not already have a recorded social-family link. It uses the surrounding POLE context, such as shared contacts, addresses, phones, and crime-location structure, to suggest possible missing associations for human review only.

  \vspace{0.25cm}
  \begin{columns}[T]
    \column{0.48\textwidth}
      \textbf{Why we chose this:}
      \begin{itemize}
        \item 1,180 raw social-family links collapse into 586 unique pair labels, not 55 \tinycode{PARTY\_TO} labels.
        \item The output is safer: possible association, not a crime claim.
        \item Social graph has meaningful community structure.
      \end{itemize}

    \column{0.48\textwidth}
      \textbf{Context graph}
      \begin{itemize}
        \item Person, Phone, Email, Location, Crime.
        \item Social links, address links, contact links.
        \item PARTY\_TO and OCCURRED\_AT retained only as context.
      \end{itemize}
  \end{columns}
\end{frame}

\begin{frame}{Supervised GDS Pipeline: Graph and Features}

  \vspace{0.12cm}
  \begin{table}
    \centering
    \begin{tabular}{p{0.28\textwidth}p{0.68\textwidth}}
      \toprule
      \textbf{Stage} & \textbf{Choice} \\
      \midrule
      Projection & A projection is the temporary in-memory graph GDS trains on. Ours includes Person/Crime/Location/Phone/Email context, with the supervised target handled separately for train/test splitting. \\
      Target & Unified undirected social-family Person--Person pairs derived from raw social-family relationship evidence. Supervised means the model learns from examples of known links. \\

      \bottomrule
    \end{tabular}
  \end{table}

\begin{frame}{Supervised GDS Pipeline: Model and Gate}
  \small
  The model is evaluated on held-out links, then used only to rank review candidates.

  \vspace{0.12cm}
  \begin{table}
    \centering
    \begin{tabular}{p{0.28\textwidth}p{0.68\textwidth}}
      \toprule
      \textbf{Stage} & \textbf{Choice} \\
      \midrule
      Model choice & Logistic Regression was tested as a simple baseline; the final pipeline keeps Random Forest (RF), a tree-ensemble model, because its scores had usable separation. \\
      Evaluation & GDS reports train, validation, and held-out test AUCPR. Held-out test means data kept aside until final evaluation. \\
      Metric & AUCPR means area under the precision-recall curve. Precision asks how many suggested links are right; recall asks how many true links are found. \\
      Deployment & Deployment means using model scores to create review candidates. The gate requires acceptable held-out AUCPR and probability spread. \\
      \bottomrule
    \end{tabular}
  \end{table}
\end{frame}

\begin{frame}[fragile]{Key ML Query}
  \small
  Candidate relationships are written only after the held-out score and score spread pass a simple gate.

  \vspace{0.15cm}
  \begin{verbatim}
CALL gds.beta.pipeline.linkPrediction.predict.stream(
  'socialFamilyContextGraph',
  {modelName: 'social-family-lp-model', topN: 200}
)
YIELD node1, node2, probability
WITH gds.util.asNode(node1) AS p1,
     gds.util.asNode(node2) AS p2, probability
WHERE NOT (p1)-[:KNOWS|KNOWS_SN|KNOWS_PHONE|KNOWS_LW|FAMILY_REL]-(p2)
RETURN p1.name, p2.name, probability
ORDER BY probability DESC;
  \end{verbatim}
\end{frame}

\begin{frame}{Comparative Insight}
  \begin{columns}[T]
    \column{0.48\textwidth}
      \textbf{Conventional ML framing}
      \begin{itemize}
        \item Needs labelled rows.
        \item Struggles when the target has only 55 positives.
        \item Harder to explain multi-hop context.
      \end{itemize}

    \column{0.48\textwidth}
      \textbf{Graph ML framing}
      \begin{itemize}
        \item Uses paths, neighbours, communities, and structural similarity.
        \item Can produce human-readable explanations.
        \item Still needs calibration before deployment.
      \end{itemize}
  \end{columns}

  \vspace{0.3cm}
  Overall comparison: GDS is strongest here as an investigative prioritization layer, not as an automated decision engine.
\end{frame}


\begin{frame}{Limitations and Ethics}
  \begin{itemize}
    \item Sparse labels make direct person-crime prediction unreliable as a deployed target.
    \item Historical hotspots are not causal forecasts.
    \item Address exposure and social proximity are context only.
    \item The model writes no automated crime claims.
    \item Human review is required before any investigative action.
    \item Final claims are backed by the saved Cypher scripts and result files.
  \end{itemize}
\end{frame}
